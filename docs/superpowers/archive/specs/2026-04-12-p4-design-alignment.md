# P4: 设计稿对齐（明显差异修复）

## 目标
对比 .pen 设计稿与实际前端，修复布局、颜色、间距、缺失组件等明显差异。不做像素级对齐。

## 设计稿来源
`pencil-new.pen` — 包含7个页面的完整UI设计

## 对比范围

| 设计稿页面 | 前端路由 | 主要检查项 |
|------------|---------|-----------|
| 01-登录页 | /login | 渐变背景色/表单位置/品牌标题/版本信息 |
| 02-Dashboard工作台 | /dashboard/overview | 侧边栏220px/统计卡片布局/图表位置/最近活动 |
| 03-模型管理-模型列表 | /model/list | 筛选栏布局/模型卡片样式/状态标签颜色/分页 |
| 05-部署监控 | /model/deployments | 监控指标卡片/时序图表/告警规则表格 |
| 10-患者列表 | /data/cdr/patients | 高级筛选区/表格列宽/导出按钮位置 |
| 11-研究项目 | /data/rdr/projects | 项目卡片网格/类型标签/创建按钮 |
| 20-用户管理 | /system/users | 角色筛选下拉/状态指示器/权限管理界面 |

## 对比方法
1. 使用 pencil MCP 导出 .pen 中每个页面的截图
2. 使用 chrome-devtools 截取前端对应页面
3. 逐页面对比，记录差异清单
4. 按影响程度排序修复

## 修复原则
- 布局结构差异（侧边栏宽度、内容区间距）— 必修
- 颜色/字号/圆角等视觉差异 — 修明显的不一致
- 缺失的UI元素（按钮、标签、图标）— 必修
- 动效/过渡/阴影等细节 — 不修
- 像素级对齐 — 不修

## 涉及文件
根据差异清单确定，预计涉及各页面 .vue 文件的 template + style 部分。

## 成功标准
- 7个页面与设计稿无明显布局差异
- 侧边栏、header、内容区结构与设计稿一致
- 状态标签、按钮样式、表格列与设计稿匹配
